What Advanced Calendar Does

Turn Jira Issues into a Visual Calendar

Advanced Calendar transforms your Jira data into an intuitive, visual schedule that makes planning and tracking effortless.

Key Features

  • Weekly and monthly views with true hourly timelines
  • Continuous multi-day bars for long-running work
  • Color-coded statuses and visit-type icons with legends
  • Site/location filter to focus on specific places or teams
  • Click-to-details modal with a "View in Jira" button
  • Live analytics showing total visits and busiest day

Use Cases

Perfect for:

  • Service visits and on-site work
  • Inspections and maintenance
  • Any time-based schedule in Jira

Opening Advanced Calendar

How to Open

  1. In Jira, go to the top navigation and click Apps.
  2. Choose Advanced Calendar for Jira (or Advanced Calendar) from the list.
  3. The calendar opens, usually in Monthly view by default.

If You Don't See the App

  • Ask your Jira admin to install Advanced Calendar for Jira from the Atlassian Marketplace.
  • Make sure you have access to the projects that the calendar uses.

Using the Calendar

Switching Between Week and Month View

Advanced Calendar has two main views:

  • Weekly view – shows an hourly timeline from Monday to Sunday
  • Monthly view – shows a classic month grid with continuous bars for multi-day issues

To switch: Look for the Week / Month toggle at the top of the calendar.

Moving Around the Calendar

Use the Prev and Next buttons to move backward or forward in time:

  • Weekly view: Each step moves you by one week (Monday–Sunday)
  • Monthly view: Each step moves you by one calendar month

The app fetches a buffered range of dates behind the scenes, so multi-day issues appear as one continuous bar even if they start before or end after the current view.

Understanding Colors, Icons, and Legends

Each bar on the calendar represents a Jira issue:

  • Bar color: comes from the issue's status category (To Do, In Progress, Done, etc.)
  • Visit-type icon or initials: comes from your Visit Type field
  • Bar position and length: comes from Start Time and End Time fields

Check the legends:

  • Status Legend: See what each color means
  • Visit Type Legend: See what each icon or initial stands for
  • Hover or click: See exact start/end times and mapped fields

Filtering by Site/Location

How to Filter

  1. Find the Site/Location dropdown above the calendar.
  2. Choose a specific site/location to show only issues from that location.
  3. Choose All (or clear the filter) to see everything again.

Filter Effects

When you change the filter:

  • The calendar updates to show only matching issues
  • The analytics cards (Total Visits, Busiest Day) update to match the filtered data

Opening Details and Working with Issues

How to Open Details

  1. Click on a bar on the calendar.
  2. A details modal opens with the key fields mapped by your admin.

What You Can Do

Inside the details modal:

  • View key fields like summary, customer, contact, assignee, site, visit type, and more
  • Click "View in Jira" to work directly in Jira
  • In Jira, you can:
    • Transition the issue (start, complete, close)
    • Add comments or attachments
    • Adjust dates and fields that control calendar appearance
  • After changes in Jira: Refresh the calendar page or move to another date and back to reload the data

Using the Analytics Cards

Quick Statistics

At the top of the calendar you'll see:

  • Total Visits – the number of issues shown in the current view and filter
  • Busiest Day – the day with the highest number of visits

When Analytics Update

These cards update automatically when you:

  • Switch between Weekly and Monthly
  • Move to a different date range
  • Change the Site/Location filter

Use Cases

Answer questions like:

  • "How many visits do we have this week?"
  • "Which day is the busiest for this month at a specific site?"
  • "How has my schedule changed compared to last week?"

Admin Panel (for reference)

Accessing the Admin Panel

  1. In Jira, click Apps → Advanced Calendar Admin.

Note: This section is primarily for admins but affects what end users see.

Admin Capabilities

In the Admin Panel you can:

  • Select projects to include on the calendar
  • Map required fields: Start Time, End Time, Site/Location, Visit Type
  • Map optional fields to show on bars or in the details modal
  • Save when you're done

Common Issues

If events are missing or look wrong on the calendar, check:

  • Wrong fields mapped in Admin
  • Issues missing Start Time or End Time
  • Insufficient permissions for projects or fields

Troubleshooting

No Events Showing

  • Check that your site/location filter is not too narrow
  • Ask an admin to verify field mappings in Advanced Calendar Admin
  • Confirm you have permissions to see the projects and issues

Wrong Start/End Times

  • Open the issue in Jira and verify the Start Time and End Time fields
  • Correct the dates/times and refresh the calendar

Multi-Day Visits Appear Broken

  • Check that Start Time and End Time cover the full range
  • Make sure you're not filtering out part of the range

Layout Issues or Refresh Problems

  • Try a hard refresh in your browser (Ctrl+Shift+R or Cmd+Shift+R)
  • If persistent, ask your admin to rebuild and redeploy the app

Need More Fields or Filters

  • Ask an admin to add more fields in the Admin Panel
  • Some changes may require a redeploy
